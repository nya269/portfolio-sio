EventHub — API Backend

R / 01
◆ Contexte
Projet mené en autonomie à ITIS Formation. Conception complète d'un écosystème de gestion d'événements composé de trois briques : un backend (API), un front web et une application mobile. Cette première brique constitue le cœur du système.
◆ Objectif
Concevoir et développer une API REST sécurisée capable de gérer deux types d'utilisateurs (organisateurs et participants), avec un système complet de création d'événements, d'inscription et de filtrage. Modéliser et implémenter la base de données relationnelle sous-jacente.
◆ Réalisation
J'ai modélisé la base (MCD puis MLD) en MySQL avec les entités Utilisateur, Événement, Inscription et Rôle. J'ai ensuite développé l'API REST en Node.js / Express : routes d'authentification, middleware de gestion des rôles, endpoints CRUD pour les événements, gestion des inscriptions et système de recherche/filtres.
◆ Résultat
API fonctionnelle, testée, capable d'alimenter simultanément les clients web et mobile. Architecture pensée pour être scalable, avec une séparation claire des responsabilités et un schéma de base de données normalisé.
◆ Stack technique
Node.js Express MySQL JWT REST
◆ Compétences SIO
B2.2 Concevoir et développer une solution applicative · B2.3 Concevoir et réaliser une base de données · B2.4 Mettre à disposition une solution applicative

EventHub — Interface Web

R / 02
◆ Contexte
Deuxième brique de l'écosystème EventHub. Une fois l'API backend opérationnelle, il était nécessaire de développer une interface web pour permettre aux utilisateurs d'interagir avec le système depuis un navigateur.
◆ Objectif
Concevoir une interface web responsive et ergonomique offrant deux espaces différenciés : un dashboard organisateur pour créer et gérer ses événements, et un espace participant pour rechercher, filtrer et s'inscrire aux événements.
◆ Réalisation
Développement en React avec une architecture par composants. Mise en place du routage, de la gestion d'état, des appels à l'API via Fetch, et d'une interface responsive en CSS. J'ai implémenté les écrans d'authentification, de création/édition d'événements, de listing avec filtres, et le formulaire d'inscription.
◆ Résultat
Interface web fonctionnelle, responsive (compatible mobile et desktop), entièrement connectée à l'API backend. L'expérience utilisateur est différenciée selon le rôle de l'utilisateur connecté.
◆ Stack technique
React JavaScript HTML5 CSS3 Fetch API
◆ Compétences SIO
B2.2 Concevoir et développer une solution applicative · B2.4 Mettre à disposition une solution applicative

EventHub — Application Mobile

R / 03
◆ Contexte
Troisième brique de l'écosystème EventHub. Étendre l'utilisation de la plateforme aux usages mobiles, en proposant une application disponible sur Android et iOS sans dupliquer le code.
◆ Objectif
Développer une application mobile cross-platform offrant les fonctionnalités essentielles aux deux types d'utilisateurs : authentification, consultation de la liste des événements, et inscription/désinscription en mobilité.
◆ Réalisation
Choix de React Native avec Expo Go pour mutualiser le code entre Android et iOS et accélérer les itérations. Implémentation des écrans d'authentification, du listing des événements, et du flux d'inscription. Connexion à l'API REST EventHub pour partager les données avec le client web.
◆ Résultat
Application mobile fonctionnelle, testée sur Expo Go, capable de communiquer avec le même backend que le frontend web. L'écosystème EventHub est désormais accessible depuis le web ET le mobile, avec une base de données unifiée.
◆ Stack technique
React Native Expo JavaScript REST API
◆ Compétences SIO

Affiches Promotions — Carrefour

R / 04
◆ Contexte
Stage en entreprise effectué chez Carrefour, au sein du service décoration. Le magasin avait besoin de supports visuels percutants pour mettre en avant les opérations promotionnelles auprès des clients.
◆ Objectif
Concevoir des affiches promotionnelles claires, attractives et conformes à la charte graphique Carrefour, destinées à être imprimées et installées dans les rayons du magasin.
◆ Réalisation
Plus de 10 affiches conçues sur Canva, en respectant la charte visuelle de l'enseigne (couleurs, typographies, codes graphiques). Travail en autonomie avec validation de mon tuteur avant impression.
◆ Résultat
Affiches imprimées et installées en rayon, validées par le service décoration. Mission qui m'a permis de développer mon sens du détail visuel et ma capacité à respecter des contraintes graphiques strictes — compétences transposables au design d'interfaces.
◆ Outils utilisés
Canva Charte graphique Mise en page
◆ Compétences mobilisées
Communication visuelle · Respect d'une charte graphique · Travail en autonomie · Sens du détail · Adaptation aux contraintes métier
