EventHub — API Backend

R / 01
◆ Contexte
Projet mené en autonomie à ITIS Formation. Conception complète d'un écosystème de gestion d'événements composé de trois briques : backend (API), front web et application mobile. Cette brique constitue le cœur du système.
◆ Objectif
Concevoir et développer une API REST sécurisée capable de gérer deux types d'utilisateurs (organisateurs et participants), avec un système complet de création d'événements, d'inscription et de filtrage, ainsi que la base de données relationnelle sous-jacente.
◆ Réalisation
Modélisation de la base (MCD puis MLD) en MySQL avec les entités Utilisateur, Événement, Inscription et Rôle. Développement de l'API REST en Node.js / Express : routes d'authentification, middleware de gestion des rôles, endpoints CRUD, système de recherche et filtres.
◆ Résultat
API fonctionnelle et testée, capable d'alimenter simultanément les clients web et mobile. Architecture scalable avec séparation claire des responsabilités et schéma de base de données normalisé.
◆ Stack technique
Node.jsExpressMySQLJWTREST
◆ Compétences mobilisées
Conception d'API REST · Modélisation de base de données relationnelle · Authentification et gestion des rôles · Architecture logicielle

EventHub — Interface Web

R / 02
◆ Contexte
Deuxième brique de l'écosystème EventHub. Une fois l'API backend opérationnelle, il était nécessaire de développer une interface web pour permettre aux utilisateurs d'interagir depuis un navigateur.
◆ Objectif
Concevoir une interface web responsive et ergonomique offrant deux espaces différenciés : un dashboard organisateur pour créer et gérer ses événements, et un espace participant pour rechercher, filtrer et s'inscrire.
◆ Réalisation
Développement en React avec une architecture par composants. Mise en place du routage, de la gestion d'état, des appels à l'API via Fetch, et d'une interface responsive en CSS. Écrans d'authentification, création/édition d'événements, listing avec filtres, formulaire d'inscription.
◆ Résultat
Interface web fonctionnelle, responsive (compatible mobile et desktop), entièrement connectée à l'API backend. L'expérience utilisateur est différenciée selon le rôle connecté.
◆ Stack technique
ReactJavaScriptHTML5CSS3Fetch API
◆ Compétences mobilisées
Développement frontend React · Intégration responsive · Consommation d'API REST · Expérience utilisateur

EventHub — Application Mobile

R / 03
◆ Contexte
Troisième brique de l'écosystème EventHub. Étendre la plateforme aux usages mobiles, en proposant une application disponible sur Android et iOS sans dupliquer le code.
◆ Objectif
Développer une application mobile cross-platform offrant les fonctionnalités essentielles : authentification, consultation de la liste des événements, et inscription/désinscription en mobilité.
◆ Réalisation
Choix de React Native avec Expo Go pour mutualiser le code entre Android et iOS. Implémentation des écrans d'authentification, du listing des événements et du flux d'inscription. Connexion à l'API REST EventHub.
◆ Résultat
Application mobile fonctionnelle testée sur Expo Go, communicant avec le même backend que le frontend web. L'écosystème EventHub est accessible web ET mobile avec une base de données unifiée.
◆ Stack technique
React NativeExpoJavaScriptREST API
◆ Compétences mobilisées
Développement mobile cross-platform · React Native & Expo · Intégration d'API REST · Adaptation Android / iOS

Maintenance Traceur Canon

R / 05
◆ Contexte
Pendant mon alternance chez Carrefour, le magasin disposait d'un traceur grand format Canon utilisé pour imprimer les affiches promotionnelles. Cet équipement devait rester opérationnel en permanence pour ne pas bloquer la chaîne de communication visuelle du magasin.
◆ Objectif
Assurer le bon fonctionnement du traceur Canon au quotidien : suivre les impressions via le logiciel RIP Queue, gérer les consommables, et intervenir en cas de problème courant pour garantir la continuité de service.
◆ Réalisation
Prise en main du logiciel RIP Queue (gestion de la file d'impression). Suivi des stocks de papier blanc et papier jaune, remplacement des rouleaux. Vérifications régulières, dépannage de premier niveau (bourrages, erreurs d'impression) et alerte au service technique en cas de panne complexe.
◆ Résultat
Traceur opérationnel en continu pendant toute la durée de l'alternance, sans interruption majeure. Le service décoration a pu produire ses affiches sans rupture, et j'ai acquis des réflexes de maintenance informatique transposables à d'autres équipements.
◆ Stack technique
Traceur CanonRIP QueueConsommablesDiagnostic
◆ Compétences mobilisées
Maintenance d'un équipement informatique · Utilisation d'un logiciel métier · Gestion des consommables · Diagnostic et dépannage de premier niveau · Sens des responsabilités
